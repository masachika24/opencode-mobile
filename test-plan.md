## 1. テスト戦略

### 1.1 テストレベル
1. **単体テスト (L1)**: 個別機能の CSS セレクタと JS 関数の動作確認
2. **統合テスト (L2)**: 複数機能の組み合わせによる相互作用の確認
3. **システムテスト (L3)**: 実際の OpenCode Web UI 上でのエンドツーエンド確認

### 1.2 テスト環境

| 環境 | 条件 |
|------|------|
| デスクトップ Chrome | Windows 11, Chrome 最新版, DevTools モバイルエミュレーション |
| モバイル実機 (優先) | iOS Safari / Android Chrome |
| デスクトップ Firefox | Windows 11, Firefox 最新版, レスポンシブデザインモード |

### 1.3 エミュレーション設定
- **iPhone SE**: 375 x 812 (iOS Safari)
- **iPhone 14 Pro**: 393 x 852 (iOS Safari)
- **Pixel 7**: 412 x 915 (Android Chrome)
- **iPad Mini**: 768 x 1024 (iOS Safari) — タブレット判定確認用

## 2. テストケース

### TC-01: ビューポート高さの安定化
| 項目 | 内容 |
|------|------|
| 関連要件 | FR-01 |
| 前提条件 | iPhone SE エミュレーション、OpenCode にログイン済み |

### TC-03: セッション一覧のタップターゲット
| 項目 | 内容 |
|------|------|
| 関連要件 | FR-03 |
| 前提条件 | セッションが3件以上存在する |
| 手順 | 1. セッション一覧を開く 2. 各セッション項目の高さを DevTools で確認 3. タップ操作で正しくセッションが切り替わるか確認 |
| 期待結果 | 各項目が最低 44px (iOS HIG 準拠) の高さを持つこと。タップで正しく反応すること |
| 検証方法 | DevTools + 手動 |

### TC-04: アクティブセッション自動スクロール
