## 依頼サマリ
OpenCode Web UI (localhost:4000, v1.14.33) を Android スマートフォン（Firefox + Tampermonkey）で快適に使うための UserScript 開発。
Tailscale 経由で自宅PCにアクセスする前提。

## 3つの課題
1. セッション一覧が不完全 → タップターゲット拡大、自動スクロール、スワイプ開閉
2. タブが多数開く → ボトムナビゲーション（Sessions/Editor/Settings）
3. モバイル画面の縦スペース不足 → ビューポート高安定化、パディング削減、safe-area対応

## 現在のスクリプト状態 (v1.5.0)
- **nav 配置**: `document.body` 直下 + `position: fixed; bottom: 0; left: 0; right: 0; z-index: 9999`
- **イベント方式**: `onclick` インライン（Firefox 対応のため addEventListener 不使用）
- **#root パディング**: JS で動的に `style.paddingBottom = '3.5rem'` を追加（重なり防止）
- **デスクトップ復帰時**: resize ハンドラで nav/chips を remove() し paddingBottom をリセット

## 既知の問題
1. nav の位置が `position: sticky` では #root の flex レイアウトと相性が悪く上部に表示される（v1.5.0 で body + fixed に戻して対処）
2. `#root > * { padding-top: 0 !important }` が送信ボタンを消失させた（v1.3.0 で削除済み）
3. Firefox for Android の下部アドレスバーとの干渉（位置の問題は解消済みのはず）
