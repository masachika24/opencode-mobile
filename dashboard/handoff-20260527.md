## 依頼サマリ
OpenCode Web UI (localhost:4000, v1.14.33) を Android スマートフォン（Firefox + Tampermonkey）で快適に使うための UserScript 開発。Tailscale 経由で自宅PCにアクセス。

## 3つの課題
1. **セッション一覧が不完全** → タップターゲット 64px 化、アクションボタン常時表示、自動スクロール、スワイプ開閉
2. **タブ多数 / 操作困難** → ボトムナビゲーション（Sessions / Editor / Settings）+ ファイルチップ水平スクロール
3. **縦スペース不足** → `h-dvh`→`100%`、パディング削減、safe-area 対応

## 現在のバージョン
**v1.5.2** — ファイル: opencode-mobile.user.js (639行)

## v1.5.2 のアーキテクチャ
- **nav 配置**: `document.body` 直下に `appendChild`
- **位置指定**: `position: fixed; bottom: 0; left: 0; right: 0; z-index: 9999`
- **イベント方式**: `onclick` インライン + `dispatchEvent(new MouseEvent(...))`（SolidJS 合成イベント対応）
- **重なり防止**: CSS で `#root { padding-bottom: calc(3.5rem + env(safe-area-inset-bottom, 0px)) }`
- **デスクトップ復帰**: resize ハンドラで nav/chips を `remove()` し paddingBottom をリセット
- **DOM 待機**: `waitForElement()` ユーティリティ（MutationObserver ベース）

## 正しい DOM セレクタ (v1.5.2 — ロケール非依存)
OpenCode v1.14.33 の実 DOM 構造に基づく。`aria-label` は UI 言語で変わるため `data-component` / `data-icon` 属性を使用。

| ボタン | セレクタ |
|--------|---------|
| **Sessions** (サイドバー開閉) | `[data-component="icon-button"][data-icon="menu"]` |
| **Editor** (プロンプト入力) | `[data-component="prompt-input"]` |
| **Settings** | `[data-component="icon-button"][data-icon="settings-gear"]` |
