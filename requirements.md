## 1. ビジネス要件

### 1.1 背景
OpenCode は主にデスクトップでの利用を想定して設計された AI コーディングアシスタントの Web UI である。
しかし、外出先やベッドサイドなど、モバイルデバイスからもセッションの確認や簡易な操作を行いたいというニーズがある。

### 1.2 目的
OpenCode Web UI をモバイルブラウザ（主にスマートフォン、タブレット）から快適に利用できるようにする UserScript を提供する。

### 1.3 対象ユーザー
- OpenCode の既存ユーザーで、モバイルデバイスからセッションを確認・操作したい人
- Tampermonkey 等の UserScript マネージャーを導入済みの技術者

## 2. 機能要件

### FR-01: ビューポート高さの安定化
**優先度**: 高  
**説明**: モバイルブラウザのアドレスバー表示/非表示によるレイアウトのガタつきを防止する。
- `h-dvh` (動的ビューポート高) を `100%` + `min-height: 100vh` に上書き
- 画面の高さが安定し、スクロール時のコンテンツ位置が不自然に変わらない

### FR-02: 不要パディングの除去
**優先度**: 高  
**説明**: 1px の無駄なパディングを除去し、貴重な縦方向ピクセルを確保する。
- `#root` の `p-px` を `0` に上書き
- サイドバー・メインコンテンツの過剰なパディングを削減

### FR-03: セッション一覧の視認性向上
**優先度**: 高  
**説明**: モバイルの小さな画面でもセッション一覧を快適に操作できる。
- セッション項目の最小高さ拡大（`min-h-14` → `min-h-16`）
- アクションボタンの常時表示（ホバー不要）
- アクティブセッションへの自動スクロール

### FR-04: モバイルナビゲーションの提供
**優先度**: 中  
**説明**: 画面下部に固定のナビゲーションバーを追加。
- ボタン: 「セッション一覧」「エディタ」「設定」
- 高さ: 約 48px（タップターゲット最低サイズ準拠）
- 現在のコンテキストをハイライト表示
- 画面全体のレイアウトに組み込み（main 領域の下部に padding 追加）

### FR-05: ファイル/タブ管理の改善
**優先度**: 中  
**説明**: 複数ファイルを開いている際の操作性を向上。
- 開いているファイルを水平スクロール可能なチップ（pill）として表示
- アクティブなファイルをハイライト
- 各チップに閉じるボタンを付与

### FR-06: スワイプジェスチャー対応
**優先度**: 低  
**説明**: 画面端からのスワイプでセッション一覧を開閉可能にする。
- 左端から右へのスワイプ: セッション一覧を開く
- 右端から左へのスワイプ: セッション一覧を閉じる
- 閾値: 画面幅の 30% 以上のスワイプで発動

### FR-07: ダークモード対応
**優先度**: 高  
**説明**: OpenCode のダークモード切替に追従し、注入する CSS も適切に配色を変更する。
- `[data-color-scheme="dark"]` セレクタを使用
- ボトムナビゲーションの背景色をテーマに合わせる

### FR-08: ノッチ・セーフエリア対応
**優先度**: 中  
**説明**: iPhone X 以降のノッチ付きデバイスでコンテンツが隠れないよう、`safe-area-inset-*` を考慮したパディングを適用する。

## 3. 非機能要件

### NFR-01: パフォーマンス
- UserScript の処理によるページ読み込み遅延は 100ms 未満
- MutationObserver のコールバック処理は 16ms 以内（60fps 維持）

### NFR-02: 互換性
- Chromium 系ブラウザ（Chrome, Edge, Brave）+ Tampermonkey
- Firefox + Greasemonkey
- iOS Safari + Userscripts アプリ（参考対応）

### NFR-03: 可逆性
- UserScript を無効化するだけで元の UI に完全に戻ること
- グローバルな副作用を残さないこと（`GM_addStyle` で注入した CSS はスクリプト無効化時に自動解除）

### NFR-04: 保守性
- 単一ファイル（`opencode-mobile.user.js`）で完結
- セクションコメントで役割を明示
- ES2020+ のモダン JavaScript を使用

## 4. 制約

| 項目 | 内容 |
|------|------|
| 動作環境 | ブラウザ + UserScript マネージャー |
| 対象URL | `http://localhost:4000/*` |
| 認証 | Basic 認証（UserScript の動作には影響なし） |
| 対象フレームワーク | React 18 + Tailwind CSS + Vite |
| 通信 | 外部通信なし（完全ローカル動作） |
| 依存ライブラリ | なし（vanilla JS のみ） |

## 5. 受け入れ基準

| ID | 基準 | 検証方法 |
|----|------|----------|
| AC-01 | モバイル表示（375x812）で全セッションが画面内にスクロール表示可能 | DevTools モバイルエミュレーション |
| AC-02 | セッション項目のタップで正しくセッションが切り替わる | 手動テスト |
| AC-03 | ボトムナビゲーションの全ボタンが正しく機能する | 手動テスト |
| AC-04 | ダークモード切替時に注入 UI の配色が適切に変化する | 手動テスト |
| AC-05 | アドレスバーの表示/非表示でコンテンツ位置がガタつかない | 手動テスト（iOS Safari 実機） |
| AC-06 | UserScript 無効化で元の UI に完全に戻る | 手動テスト |
| AC-07 | コンソールにエラーが出力されない | DevTools Console |

---

## 6. 対象外

- オフライン対応
- プッシュ通知
- マルチタッチジェスチャー（ピンチズーム等はブラウザ標準動作に任せる）
- 音声入力
- PWA 化（Service Worker, manifest.json）
- サーバーサイドの改変
- デスクトップモードでの表示変更（1024px 以上は元の UI を維持）

---

## 7. 用語定義

| 用語 | 定義 |
|------|------|
| UserScript | Tampermonkey/Greasemonkey で実行されるユーザー作成スクリプト |
| モバイル | 画面幅 1024px 未満のデバイス（スマートフォン・小型タブレット） |
| dvh | Dynamic Viewport Height。モバイルブラウザの UI 変化に応じて変動するビューポート高の単位 |
| セーフエリア | iPhone X 以降のノッチやホームインジケーターを避けた安全な表示領域 |
